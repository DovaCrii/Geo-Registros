# opencode-handoff

## Objetivo
Preparar instrucciones claras para que OpenCode ejecute la tarea principal.

## Cuándo usarla
- Cuando Codex ya revisó o planificó una tarea.
- Cuando se necesita traspasar contexto sin duplicar memoria.

## Qué hace
- Resume objetivo, alcance y restricciones.
- Lista archivos probables y criterio de aceptación.
- Aclara qué debe evitarse.

## Límites
- No reescribe el plan completo si no hace falta.
- No asume rol de orquestador.
- No modifica archivos.

## Resultado esperado
- Handoff breve.
- Instrucciones accionables.
- Restricciones explícitas para OpenCode.
