# codex-project-review

## Objetivo
Revisar arquitectura, riesgos, estructura y consistencia del proyecto sin editar código.

## Cuándo usarla
- Antes de una implementación importante.
- Cuando haya dudas sobre arquitectura, dependencias o coherencia entre documentos y código.
- Cuando se necesite una lectura crítica del estado del repositorio.

## Qué hace
- Inspecciona estructura, convenciones y puntos de acoplamiento.
- Detecta riesgos, contradicciones y deuda técnica.
- Resume hallazgos accionables para OpenCode o para una revisión humana.

## Límites
- No modifica archivos.
- No instala dependencias.
- No toca Engram ni configuración ajena.

## Resultado esperado
- Lista corta de hallazgos.
- Riesgos priorizados.
- Recomendación de siguiente paso.
