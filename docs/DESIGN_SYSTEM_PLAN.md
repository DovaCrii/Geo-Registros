# DESIGN_SYSTEM_PLAN.md

## Direccion visual
Base clara, profesional y operacional. Dark mode queda como modo secundario. La interfaz debe sentirse confiable para operaciones tecnicas y suficientemente elegante para demo comercial.

## Reglas base
- Texto principal 15-16px.
- Labels 13-14px.
- Evitar textos demasiado pequenos.
- Cards limpias, bordes suaves y buen contraste.
- Botones comodos y claramente accionables.
- Badges de estado con color y texto.
- No depender solo del color para comunicar estado.
- Controles de mapa visibles y faciles de reconocer.

## Componentes prioritarios
- Boton primario, secundario y destructivo.
- Card operacional.
- Badge de estado.
- Timeline de permiso.
- Checklist documental.
- Panel de siguiente accion.
- Empty states honestos.

## Implementacion sugerida
Primero documentar tokens y variantes. Luego aplicar por pantalla: dashboard, lista de planes, detalle de plan, wizard y editor de mapa.
