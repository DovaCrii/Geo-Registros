# codex-safe-implementation

## Objetivo
Permitir implementación puntual solo si el usuario lo solicita explícitamente.

## Cuándo usarla
- Cuando el usuario pida a Codex implementar directamente.
- Cuando el cambio sea pequeño, controlado y compatible con el contrato operativo.

## Qué hace
- Aplica cambios acotados.
- Mantiene el alcance mínimo necesario.
- Conserva seguridad y trazabilidad.

## Límites
- Solo se usa con autorización explícita del usuario.
- No toca Engram.
- No convierte a Codex en la herramienta principal.

## Resultado esperado
- Cambio pequeño, verificable y documentado.
- Validación posterior con QA.
